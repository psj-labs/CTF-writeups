# KubSTF CTF - capyblog Writeup

## 개요

- 본 문제는 웹 애플리케이션 내부에 존재하는 system() 함수 호출을 이용한 Command Injection 취약점이 존재하는 파일을 통해 서버 내부 파일을 탐색하고 flag를 획득하는 문제입니다.
- 목표는 서버 내부 경로를 탐색하여 flag 파일을 찾아내는 것입니다.

---

### 메인 화면 확인

- 다음은 문제 메인화면입니다.  
- 회원가입 및 로그인 상태입니다.

![32](/Web/imgs01/32.png)

### 취약점 탐색

- 댓글 기능이 존재하여 XSS를 시도했지만 동작하지 않습니다.
  (사진을 캡쳐받지 못했습니다..)

---

### robots.txt 확인

- /robots.txt를 확인합니다.

- data 디렉토리가 존재하는 것을 확인합니다.
  (사진을 캡쳐받지 못했습니다..)

### 디렉토리 접근 시도

- /data로 접근하면 403 Forbidden이 발생합니다.
   (사진을 캡쳐받지 못했습니다..)

---

### 디렉토리 브루트포싱

- 추가 경로를 찾기 위해 ffuf를 사용합니다.
```
ffuf -u "http://193.42.127.24/FUZZ" -w /usr/share/wordlists/seclists/Discovery/Web-Content/common.txt -mc 200,301,302 -e .php,.txt,.html
```
![33](/Web/imgs01/33.png)

- 여러 경로 중 shell.php를 발견합니다.

---

### 취약 엔드포인트 접근

- shell.php로 접근합니다.

![34](/Web/imgs01/34.png)

- 에러 메시지가 출력됩니다.

> GET 파라미터 c가 전달되지 않아 system() 함수에 빈 값이 들어가 오류 메시지가 출력된 상태입니다.

---

### 취약점 분석

- system() 함수가 존재합니다.

> 외부 입력을 통해 명령어 실행 가능하다고 판단합니다.

---

### 명령어 실행 테스트

- c 파라미터에 ls를 전달합니다.
```
?c=ls
```
![35](/Web/imgs01/35.png)

- 현재 디렉토리 목록이 출력됩니다.

---

### 서버 내부 탐색

- 웹 루트 경로로 이동합니다.
```
/var/www/html/data
```
![36](/Web/imgs01/36.png)

> flag.txt 파일을 발견합니다.

---

### flag 획득

- 파일 내용을 출력합니다.
```
cat /var/www/html/data/flag.txt
```
![37](/Web/imgs01/37.png)

> flag를 획득합니다.
